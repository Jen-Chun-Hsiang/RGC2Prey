# Recent experiment update: direct-image control, pixelized-temporal-filter control, and white-spot on black background testing

## Purpose of this document

This document summarizes the **recent experimental update** in manuscript-ready language. It is written for readers who need to understand the **processing pipeline, stimulus design, control conditions, and evaluation logic**, without needing to read the source code.

The update introduces three linked changes:

1. A **direct-image control** condition, in which the retinal spatial filter bank is bypassed.
2. A **pixelized temporal-filter control** condition, in which temporal filtering is reduced to a one-frame impulse.
3. A **test-only stimulus substitution**, in which the usual natural-image blend background is replaced by a **pure black background**, and the cricket image is replaced by a **single white spot**.

These conditions were designed to isolate which parts of the pipeline are responsible for behavioral decoding performance:

- spatial retinal filtering,
- temporal retinal filtering,
- naturalistic object shape/texture,
- and natural-image background structure.

---

## High-level experimental logic

The core task remains the same across all conditions:

- generate a moving object trajectory and a moving background trajectory,
- synthesize a binocular or monocular movie sequence,
- transform the movie into a time-varying neural-like representation,
- decode object position over time with a CNN-LSTM network,
- compare performance across stimulus and encoding variants.

The new controls were introduced to answer the following questions:

### 1. Direct-image control
Can the downstream network localize the target when it receives **temporally filtered image pixels directly**, rather than a mapped retinal ganglion cell (RGC) population response?

### 2. Pixelized temporal-filter control
Can the decoder localize the target when the retinal pathway has **spatial selectivity** but **no temporal integration beyond the current frame**?

### 3. White-spot / black-background test
How well does the trained model generalize when the test stimulus is reduced to a **single bright spot** on a **uniform black background**, removing natural image structure from both foreground and background?

---

## Shared stimulus generation pipeline

Across all of these updated experiments, each training or test sample is generated as follows.

### 1. Object and background motion
Two independent trajectories are generated:

- an **object trajectory** for the top image,
- a **background trajectory** for the bottom image.

Both trajectories are stochastic and are governed by:

- a stay-vs-move transition process,
- initial velocity,
- momentum decay,
- random velocity perturbations,
- random turning angle,
- and a fixed motion boundary.

The trajectories are truncated to the same duration, then padded with an initial static segment so that each sequence begins with a stationary pre-motion period.

### 2. Dynamic object scaling
The object is rescaled over time to mimic approach/recession. For each sequence:

- the object scale begins at a lower value,
- increases toward a larger value,
- and can be jittered by a dynamic scaling term.

This scaling factor is also used in binocular mode to determine disparity.

### 3. Binocular rendering
For binocular experiments, a left-eye and right-eye object position are generated by introducing a disparity shift derived from object scale or from a fixed disparity when that option is used.

The final synthesized movie contains two channels when binocular rendering is enabled, corresponding to the two eye-specific views.

### 4. Image composition
Each movie frame is made by overlaying the object image onto the background image.

Important implementation details relevant for reproducibility:

- the object is alpha-composited onto the background,
- only the **green image channel** is retained for the final movie representation,
- the frame is cropped to the target field of view,
- and pixel values are scaled to the range $[-1, 1]$ before entering the neural encoding stage.

Thus, although the source files are RGB/RGBA images, the effective stimulus delivered to the model is a **single-channel green-intensity movie**.

## Shared network readout

All updated conditions use the same downstream decoder family:

- a convolutional feature extractor applied frame by frame,
- followed by an LSTM over time,
- followed by linear projection to 2D object coordinates.

For the recent runs documented here, the decoder configuration was:

- CNN extractor version: **4**
- CNN feature dimension: **256**
- LSTM hidden size: **128**
- LSTM layers: **4**
- input normalization: **enabled**
- channel-wise normalization: **enabled**
- sequence reshaping for efficient batched CNN processing: **enabled**

The model predicts object $(x,y)$ position at every time step.

## Condition A: direct-image control

### Functional definition
In the direct-image control, the retinal spatial filter bank is **bypassed**.

Instead of:

1. projecting each movie frame onto many spatial receptive fields,
2. applying temporal filters to each RGC,
3. and mapping those RGC signals to a fixed grid,

the pipeline does the following:

1. take the movie itself,
2. apply a **shared temporal filter directly to every pixel**, independently,
3. resize the filtered image to the model input grid,
4. and feed this image sequence to the CNN-LSTM decoder.

### What this condition preserves
The direct-image control still preserves:

- the same object and background motion statistics,
- the same binocular rendering when binocular mode is enabled,
- the same cropping and scaling pipeline,
- the same CNN-LSTM decoder,
- and the same target coordinate labels.

### What this condition removes
This control removes the explicit retinal population encoding step:

- no spatial receptive-field bank,
- no sparse RGC center locations,
- no mapping from irregular RGC positions to a fixed retinotopic grid,
- and no center-of-mass baseline derived from RGC coordinates.

### Interpretation
This condition tests whether performance depends on the **retinal spatial encoding architecture** itself, or whether comparable localization can be obtained from a simpler temporally filtered image representation.

---

## Condition B: pixelized temporal-filter control

### Functional definition
In the pixelized temporal-filter control, the temporal kernel is replaced by a **delta-like one-frame impulse**:

- the temporal filter is all zeros except the final sample,
- effectively making the response depend only on the current frame,
- with no integration over earlier frames inside the retinal front end.

### What this means in practice
Under this condition, each model unit still receives a **spatially filtered** input, but the temporal component is reduced to an instantaneous readout.

Therefore:

- spatial selectivity is preserved,
- grid mapping is preserved,
- binocular channel structure is preserved,
- but multi-frame temporal filtering within the retinal encoder is removed.

### Interpretation
This control isolates the value of **retinal temporal integration**. If performance drops relative to the standard temporal-filter condition, that drop can be attributed to the loss of temporal smoothing / temporal feature extraction prior to the CNN-LSTM decoder.

---

## Optional combined condition: direct image + pixelized temporal filter

A related ablation also combines both modifications:

- **direct image** input,
- plus a **delta temporal kernel**.

In that combined setting, the front end becomes approximately a resized current-frame image pathway, with no retinal spatial filtering and no temporal integration before the decoder.

This variant is useful as a lower-bound control on the contribution of the engineered retinal front end.

## Updated test condition: white spot on pure black background

## Functional change at test time
The recent evaluation update replaces the standard naturalistic test inputs with a minimal stimulus:

- **background**: `black-bg` folder, containing only pure black image(s),
- **object**: `white-spot` folder, containing a single white-spot image.

This substitution is applied **at test time**, while keeping the rest of the motion-generation and decoding pipeline unchanged.

### What remains variable at test time
Even in the white-spot / black-background condition, the following variables still change from sample to sample:

- object trajectory,
- background trajectory,
- object scale over time,
- binocular disparity in binocular mode,
- and any explicitly requested test noise level.

### What is removed by this test condition
This test condition removes several naturalistic cues:

- natural background texture,
- natural foreground texture,
- shape variation across multiple cricket exemplars,
- and image-specific cricket body/head geometry.

### Important coordinate-handling consequence
Because the white-spot object is not one of the cricket exemplars with a body/head correction table, the object center is evaluated directly from the rendered stimulus geometry.

In practice, this means:

- no cricket-specific coordinate correction is applied,
- there is no dependence on per-image body/head landmark metadata,
- and the target location corresponds to the rendered spot position itself.

### Why this matters scientifically
This testing regime isolates whether the learned decoder depends on:

- cricket-specific shape cues,
- natural-scene background statistics,
- or instead on more generic motion- and position-related information.

## Updated evaluation protocol

For the recent evaluation update, model checkpoints can be tested with the following substitutions:

- replace the test background folder with **black-bg**,
- replace the test object folder with **white-spot**.

This produces a minimal stimulus regime with a bright spot moving over a uniform black field.

## Recommended reporting language for evaluation

A clear Methods description is:

> For the stimulus-substitution test, we retained the trained model weights and motion-generation procedure but replaced the standard natural-image testing inputs with a single white-spot object rendered over a pure black background. This manipulation preserved trajectory statistics, scale dynamics, and binocular geometry, while removing naturalistic object texture and background structure.

## What to report in the paper for this condition

At minimum, report:

- which checkpoint/condition was evaluated,
- whether testing used `white-spot`, `black-bg`, or both,
- whether binocular input was preserved,
- whether disparity was natural or fixed,
- and what noise level(s) were used during testing.

If multiple test noise levels were used, state the exact sweep in the figure legend.

## Practical notes for interpretation

1. **Direct-image and pixelized-TF are not the same manipulation.**
   - Direct-image removes the retinal spatial population code.
   - Pixelized-TF removes temporal integration while keeping spatial encoding.

2. **White-spot / black-background is a test-set manipulation, not necessarily a retraining condition.**
   - It should be described as a stimulus substitution at evaluation unless the model was explicitly retrained on that same simplified stimulus set.

3. **Center-of-mass baselines should not be treated as available in direct-image mode.**
   - That baseline depends on RGC center coordinates and therefore is not defined once the retinal grid is bypassed.

4. **The effective stimulus channel is the green channel.**
   - This matters for external replication, because the model is not operating on full RGB movies.

## Short version for the paper supplement

> Recent control experiments introduced two front-end ablations and one stimulus-substitution test. The direct-image ablation bypassed the retinal receptive-field bank and applied temporal filtering directly to image pixels before decoding. The pixelized temporal-filter ablation replaced the retinal temporal kernel with a one-frame impulse, preserving spatial filtering but removing temporal integration. In the simplified test condition, the natural-image background was replaced by a pure black image and the cricket foreground was replaced by a single white spot, while preserving motion statistics, scaling, and binocular geometry. These manipulations were used to separate the contributions of retinal spatial filtering, retinal temporal filtering, and natural-image stimulus structure to object-localization performance.
