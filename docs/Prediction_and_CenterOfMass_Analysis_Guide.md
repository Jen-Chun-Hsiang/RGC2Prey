# Prediction + Center-of-Mass Analysis Guide

This document describes exactly how evaluation outputs are generated by:

- `visual_preycapture_prediction.py`
- `datasets/sim_cricket.py` (`Cricket2RGCs.__getitem__`)
- `utils/file_name.py` (`make_file_name`)

It focuses on:

1. How **trained model prediction** is computed.
2. How the **updated center-of-mass (COM)** baseline is computed.
3. Exact saved file formats (`.mat`, `.npz`, `.png`, videos/frames).

---

## 1) Entry point and experiment sweep

Run script:

- `visual_preycapture_prediction.py`

CLI parser (`parse_args`) accepts:

- `--experiment_names` (required, one or more)
- `--noise_levels` (optional list)
- `--fix_disparity_degrees` (optional list)
- visualization/movie options and `--disable_center_of_mass`

Main loop behavior:

- For each `experiment_name`
  - For each `fix_disparity_degree` (if provided)
    - For each `noise_level` (or `[None]`)
      - Call `run_experiment(...)`

This means one run can produce many output files, one set per `(experiment_name, disparity, noise)` condition.

## 3) Model evaluation pipeline (trained prediction)

Inside `run_experiment(...)`:

1. **Load checkpoint**
   - Loads model weights and `args` from checkpoint via `CheckpointLoader`.
   - Applies compatibility defaults for missing legacy args.

2. **Determine COM availability policy**
   - Starts with `is_plot_centerFR = not disable_center_of_mass`.
   - Then forcibly disables COM when:
     - `args.is_direct_image == True`
     - OR `args.is_both_ON_OFF == True`
     - OR `args.is_two_grids == True`

3. **Rebuild encoding front-end**
   - Recreates `RGCrfArray` and `Cricket2RGCs` using checkpoint args.

4. **Section 1 test loss pass**
   - Builds `test_dataset` (`num_samples=1000`, `is_syn_mov_shown=False`).
   - Runs model in eval mode over `test_loader`.
   - Computes per-batch MSE loss and stores in `test_losses`.

5. **Section 2 trajectory pass for analysis**
   - Builds `analysis_dataset` (`num_samples=int(0.1 * 1000)=100`, `is_syn_mov_shown=True`).
   - For each sample, stores:
     - true path
     - model predicted path
     - background path
     - optional COM path
     - metadata (scaling factors, bg image name, image id)
   - Computes per-sample trajectory MSE for model and COM on this section.

6. **Section 3 visualization/movie pass**
   - Uses selected section-2 samples (`--visual_sample_ids`) when provided.
   - Otherwise generates fresh visualization samples.
   - Saves per-sample PNG plots and optional videos/frames.

## 4) How COM is generated (updated behavior)

COM originates in `Cricket2RGCs.__getitem__` (`datasets/sim_cricket.py`).

### 4.1 Computation

For non-direct-image mode and when `grid_coords` is provided:

```python
s_rgc_time = torch.abs(rgc_time - rgc_time[:, :1])
weighted_sum = torch.einsum('nt,nc->tc', s_rgc_time, self.grid_coords)
sum_rates = torch.sum(s_rgc_time, dim=0, keepdim=True).clamp(min=1e-6).view(-1, 1)
weighted_coords = (weighted_sum / sum_rates).detach().numpy()
```

- `weighted_coords` shape: `[T_out, 2]`
- It is a center-of-mass of RGC coordinates weighted by activity change from the first timepoint.

If COM is not available:

- returns `weighted_coords = np.array(0)`.

### 4.2 In evaluation script

`visual_preycapture_prediction.py` converts raw `weighted_coords` to XY paths via `_coerce_xy_path(...)`:

- valid COM path -> shape `[T, 2]`
- invalid/non-available COM -> replaced with `NaN` path (`np.full_like(path, np.nan)`)

Then:

- `center_valid_mask = ~np.isnan(cm_paths_for_metric).any(axis=(1, 2))`
- COM MSE computed only on valid samples.

### 4.3 Important availability rules

COM baseline is intentionally disabled in these conditions:

- `--disable_center_of_mass`
- checkpoint `is_direct_image=True`
- checkpoint `is_both_ON_OFF=True`
- checkpoint `is_two_grids=True`

So COM fields may exist in output `.mat`, but values are `NaN`/invalid when disabled.

## 5.1 MAT file A: batch test loss summary

Path:

```
{Mats}/{file_name}_{epoch_number}_prediction_error.mat
```

Keys:

- `test_losses` : `float`, shape `[num_batches]`
- `training_losses` : `float`, shape `[num_train_epochs_saved]`

Notes:

- `test_losses` here is **per-batch**, not per-sample.
- Uses Section 1 loader (`num_samples=1000`, shuffled).

## 5.2 MAT file B: trajectories + per-sample metrics

Path:

```
{Mats}/{file_name}_{epoch_number}_prediction_error_with_path.mat
```

Keys and meaning:

- `test_losses` : Section 2 per-batch losses
- `training_losses` : loaded from checkpoint
- `all_paths` : flattened true trajectories, shape `[N, 2T]`
- `all_paths_bg` : flattened background trajectories, shape `[N, 2T]`
- `all_scaling_factors` : shape `[N, T]`
- `all_bg_file` : object/string array of background image names
- `all_id_numbers` : int array of cricket image ids
- `all_paths_pred` : model predictions, typically shape `[N, 1, T, 2]` (or `[N, T, 2]` in some paths)
- `all_path_cm` : flattened COM trajectories, shape `[N, 2T]` (`NaN` where invalid)
- `model_mse_per_sample` : shape `[N]`, MSE over `(T,2)` per sample
- `center_mse_per_sample` : shape `[N]`, `NaN` for invalid COM samples
- `center_valid_mask` : uint8 shape `[N]` (1 valid, 0 invalid)
- `model_mse_mean` : shape `[1]`, `nanmean(model_mse_per_sample)`
- `center_mse_mean` : shape `[1]`, `nanmean(center_mse_per_sample)`
- `center_estimation_enabled` : uint8 shape `[1]`, 1 if COM branch enabled in this run
- `center_is_independent_from_model` : uint8 shape `[1]`, always 1
- `analysis_indices` : `np.arange(N)`

### Recommended way to compare model vs COM

Use this file, and only compare on matched valid subset:

1. Read `center_estimation_enabled`; if 0, do not plot COM as active baseline.
2. Use `center_valid_mask==1` to select samples.
3. Compare `model_mse_per_sample[mask]` vs `center_mse_per_sample[mask]`.

This avoids accidental mismatch across different sections/loaders.

## 6) Why COM can beat trained model in difficult backgrounds

Given current implementation, this is possible and not automatically a bug:

- COM is a deterministic, hand-crafted estimator from encoder activity.
- Model prediction depends on checkpoint fit + decoder generalization under current test condition.
- If model is slightly mismatched to test distribution, COM can outperform it on hard clutter.

However, ensure your figure only uses COM when:

- `center_estimation_enabled == 1`, and
- comparisons are made on `center_valid_mask == 1` samples.

---

## 7) Known analysis caveats

1. **Two evaluation sections use different datasets/loaders**
   - Section 1 (`prediction_error.mat`) and Section 2 (`prediction_error_with_path.mat`) are generated from different dataset instances.
   - Do not mix Section 1 and Section 2 metrics as if they were computed on identical samples.

2. **COM only meaningful in single-grid RGC mode**
   - It is intentionally disabled in direct-image / two-grid / ON-OFF configurations in the current script.

3. **`all_paths_pred` dimensionality can vary**
   - Code handles both `[N,1,T,2]` and `[N,T,2]` before metric computation.

---

## 8) Minimal reproducible checklist for analysis

For each condition file (`*_prediction_error_with_path.mat`):

1. Verify condition identity from filename (`experiment`, `object`, `background`, optional `disp`, optional `noise`).
2. Read `center_estimation_enabled`.
3. Read `center_valid_mask` and valid count.
4. Compute summary on matched samples:
   - model: `mean(model_mse_per_sample[mask])`
   - COM: `mean(center_mse_per_sample[mask])`
5. Report both valid sample count and means.

This gives fair, condition-correct comparison between trained model and COM baseline.

## 9) MATLAB-focused usage notes (important)

This section is specifically for MATLAB analysis of files saved by Python `scipy.io.savemat`.

### 9.1 MATLAB loading behavior you should expect

When loading `*_prediction_error_with_path.mat` in MATLAB:

- Numeric arrays are often imported with extra singleton dimensions.
- `uint8` masks/flags remain `uint8` (convert to logical for indexing).
- String/object arrays (like `all_bg_file`) may appear as cell arrays or char matrices depending on MATLAB/scipy interaction.

Recommended pattern:

```matlab
S = load(mat_file);
```

Then normalize field shapes explicitly (see template below).

### 9.2 Field-by-field MATLAB interpretation

- `model_mse_per_sample`: expected 1D length `N`; in MATLAB often `[N,1]` or `[1,N]`.
- `center_mse_per_sample`: same as above, includes `NaN` when COM invalid.
- `center_valid_mask`: `uint8`, values `0`/`1`; convert using `logical(...)`.
- `center_estimation_enabled`: scalar-ish `uint8` array; read as `double(S.center_estimation_enabled(1))`.
- `all_paths`, `all_paths_bg`, `all_path_cm`: flattened trajectories `[N, 2T]`.
- `all_paths_pred`: may come as
  - `[N,1,T,2]` (common), or
  - `[N,T,2]`.

For robust MATLAB analysis, do not assume one fixed rank for `all_paths_pred`.

### 9.3 Robust MATLAB template (copy/paste)

```matlab
S = load(mat_file);

% 1) Read enable flag safely
center_enabled = false;
if isfield(S, 'center_estimation_enabled')
   center_enabled = logical(double(S.center_estimation_enabled(1)));
end

% 2) Read per-sample MSE vectors as columns
model_mse = S.model_mse_per_sample(:);
center_mse = S.center_mse_per_sample(:);

% 3) Read valid mask safely
if isfield(S, 'center_valid_mask')
   center_valid = logical(S.center_valid_mask(:));
else
   center_valid = ~isnan(center_mse);
end

% 4) Matched-sample comparison
if center_enabled
   mask = center_valid & ~isnan(center_mse) & ~isnan(model_mse);
   model_mean = mean(model_mse(mask));
   center_mean = mean(center_mse(mask));
   n_valid = nnz(mask);
else
   mask = ~isnan(model_mse);
   model_mean = mean(model_mse(mask));
   center_mean = NaN;
   n_valid = 0;
end

fprintf('Model mean MSE: %.6f\n', model_mean);
fprintf('Center mean MSE: %.6f\n', center_mean);
fprintf('Valid COM samples: %d\n', n_valid);
```

### 9.4 Reconstructing `[N,T,2]` trajectories in MATLAB

From flattened true paths (`all_paths`, shape `[N,2T]`):

```matlab
P = S.all_paths;      % [N, 2T]
N = size(P,1);
T = size(P,2)/2;
P3 = reshape(P, [N, T, 2]);
```

For predictions (`all_paths_pred`), normalize rank first:

```matlab
Y = S.all_paths_pred;
Y = squeeze(Y);

% If shape became [T,2] for a single sample, restore batch dim
if ndims(Y)==2 && size(Y,2)==2
   Y = reshape(Y, [1, size(Y,1), 2]);
end

% If shape is [N,2,T] due to axis ordering surprises, permute to [N,T,2]
if ndims(Y)==3 && size(Y,2)==2 && size(Y,3)~=2
   Y = permute(Y, [1,3,2]);
end
```

### 9.5 MATLAB-safe handling for `all_bg_file`

Depending on MAT conversion, `all_bg_file` may be:

- cell array of char vectors, or
- char matrix.

Robust conversion:

```matlab
bg = S.all_bg_file;
if ischar(bg)
   bg_list = cellstr(bg);
elseif iscell(bg)
   bg_list = cellfun(@string, bg, 'UniformOutput', false);
else
   bg_list = {};
end
```

### 9.6 MATLAB caveat: do not mix section metrics

For fair model-vs-COM comparison in MATLAB, use only:

- `*_prediction_error_with_path.mat`

Do not mix `test_losses` from `*_prediction_error.mat` with per-sample COM metrics, because they come from different dataset sections.
